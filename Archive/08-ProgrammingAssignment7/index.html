Assignment Requirements

For this week's homework, you'll focus on adding some UI your designer wants you to implement when a user goes to upload an assignment to the web. Here's what you'll do. Starting with the provided starter project:

On the Assignment page, near the bottom, do the following:

  • Use the "Button with View" init method of the Button view like this
Button(action: {
    print("Pressed")
}) {
    HStack(spacing: 10) { 
      Image(named:"imageNameHere")
      Text("button text if needed")
    }
}
  • [GESTURES] To add a button at the bottom of the view that a user will be able to slide from left to right using a gesture.
  • [ANIMATIONS] When the user moves the button from the left side of the screen to the right (think about the original "Slide to Unlock" on iOS):
    • Return the button to the original position on screen
    • Make a Star Image appear above the area where you slid the button that:
      • Animates opacity from 0 to 1 when finished
      • Animates from a small image to a larger one
      • While it grows, spin the star via a rotation animation
      • Animate the color of the star from black to yellow
  • [NOTIFICATIONS] When the animation completes, fire a local notification letting the user know that the upload is complete, and then return the star to its original state.

A few notes:

  • The notification can fire after the animations are complete - usually this would come from a network response in a real app. Since we're not dealing with a real networking communication on the backend, you can trigger the notification to fire after a certain number of seconds.
  • To reset the state of the star after the notification has come in, you can delay those changes using DispatchQueue.main.asyncAfter(_,work:), where the first argument can be .now() + numSeconds where numSeconds is the delay from now that you want the work to happen
  • Remember, if you need to pass in values in a preview where a binding is expected, you can use .constant(value)
  • You can download the SFSymbols app from Apple to get a number of built in systemNamed symbols you can use with the Image view (using Image(systemName:) initializer
  • There is a modifier called onChange(of: perform:) that may come in handy if you want to respond to the change of a property.
  • I've crafted the starter project so that you deal with each view individually, so you can a) handle each view independently and b) take advantage of SwiftUI's composition capabilities. I strongly recommend dealing with each view separately, and then compositing them together at the end to make your overall view. Take advantage of @State properties and their ability to pass into other views via @Bindings - and remember when the @Bindings change, the @State properties will update and the views will refresh!

Assignment Submission

  • Zip up the Xcode project folder. This should be the folder that has the .xcodeproj file inside it.
  • Rename the zip to LastName_Homework#.zip
  • Upload the zip to the Programming Assignment 6 area on Blackboard BEFORE 11:59 PM ET on the day it is due (Sunday of Week 7).
  • Be sure to test your app before submitting, so you know it works as intended!

Assignment Rubrics

In addition to the normal grading criteria stated in the syllabus, you will be graded on:

  • Proper functionality of the animations
  • Proper use and behavior of gesture
  • Proper display of notification
