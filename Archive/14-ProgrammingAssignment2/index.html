Getting Ready

In your second programming assignment, you'll add on to the user interfaces that we started building in class. Please download the starter project from the following link to get started:

Downlink Week 2 Materials (link will be active start of week 2)

Assignment Requirements

To build on the project, you should do the following:

  • Make the Student SwiftUI screen with the following features:
    • The name, address, email, jhed, program, and an image at the top, above the name.
    • The average grade for the student.
    • The ability to tap on the email address and present the MailView that was discussed in class. Some hints for this one, since we didn't cover it in class - look into the following modifiers:
      • .onTapGesture() (to state what happens when a user taps on a view)
      • .sheet(isPresented: content:) (which presents views in a modal sheet over the current view; the example app code for an example of this.)
    • Update the preview struct to also show the StudentView in iPhone8 (no notch, button) and iPhone 13 (no button, notch) frames. There is a modifier that can help you with this (you job: find and use it)!
    • Present the data for the student in a Form instead of a VStack - what's different between the 2 views?
  • Inside the StudentsView.swift file, also complete the following views:
    • StudentRow should contains an image of the student and their name. Each row should be equally sized, which means you need to force the image to be the same size for all students.
    • StudentTable should display a list of all students in the class.
    • Make the Assignment SwiftUI view with the following features:
      • The assignment name as the title of the screen, using the largeTitle font
      • The type of assignment
      • The due date for the assignment (use a DateFormatter here!)
      • The max grade for the assignment
      • The description of the assignment
    • Once you complete those, the CourseClassView will show content that you'll see in the app (via the ContentView). Confirm that each tab of the ContentView shows the proper information.
      • You can use the Preview Canvas or just build the UI from code; either way you will want to use the Library in Xcode
      • Fields should have labels where appropriate (for example, don't just write the Due Date, add a label to show what it is)

        Assignment Submission

        • Zip up the Xcode project folder. This should be the folder that has the .xcodeproj file inside it.
        • Rename the zip to LastName_Homework3.zip
        • Upload the zip to the Programming Assignment 2 area on Blackboard BEFORE 11:59 PM ET on the day it is due (Sunday of Week 2).
        • Be sure to test your app before submitting, so you know it works as intended! I suggest taking your zip, opening it in a different folder on your computer, and running it as a test.

        Assignment Rubrics

        In addition to the normal grading criteria stated in the syllabus, you will be graded on:

        • The Student view should contain all required properties with a decent layout (take advantage of Forms, VStacks and HStacks)
        • The Student previews should appear as stated in the canvas
        • The Assignments view should show the requested details, with appropriate font sizes and labels in front of values so you know what you're reading.
