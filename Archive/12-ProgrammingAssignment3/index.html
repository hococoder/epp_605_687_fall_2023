Getting Ready

In your third programming assignment, you'll use UserDefaults to store some small data, and also add data into a Core Data database to help initialize the data model for your app. Please download the starter project from the following link to get started:

Downlink Homework 3 Starter Project (link will be active start of week 3)

Assignment Requirements

This week, you'll use your programming assignment to add Core Data capabilities to the app, as well as give the student the chance to enroll in classes. There's a lot to do, so I've broken things done by screen/model object.

A LOT of this is just repeating steps done for the other types, so don't panic! Also, be sure to check out the list of available SwiftUI modifiers if you're stumped on adding functionality - there may be something that you need in that list! Finally - this is not necessarily the proper order to do these things in - just a list!

  • The LoginScreen view
    • Add a method to store the value the users enters in the username field to UserDefaults.
    • When you login during your testing, be sure to use the user "Jsmith2" from the JSON data, since that is what the database gets populated with (we don't have full registration for the user in the app, so we'll just use this)
    • If the user has entered a username previously, prepopulate that in the username field on startup
  • Course Related Views
    • Finish the CompleteClassListView
      • Add a NSFetchRequest that gets the student from the database for the JHED stored in the UserDefaults
      • In the forEach block, list the names of the available classes
      • Add a button to the navigation bar titled "Enroll"
      • When a class is chosen and the Enroll button is tapped, update the database, dismiss the modal screen, and return to the enrolled list (which should now be updated!).
      • HINT 1: Put the table in edit mode to allow the rows in the table to become selectable. This can be done programmatically
      • HINT 2: A quirk in SwiftUI: on the ForEach block, add the onDelete(perform:removeRows) modifier to properly enable the selection capabilities of the row (the list has to have that functionality turned on to enable selection of rows, even with edit mode turned on, which is weird, but that is what is required!). In addition add func removeRows(at offsets: IndexSet) {} as a method to CompleteClassListView to satisfy the compiler once you had added the onDelete call
  • Finish the EnrolledCourseClassView
    • Add a NSFetchRequest that gets the student from the database for the JHED stored in the UserDefaults
    • Use a List to display the list of classes the student is enrolled in. Tapping on that row should take the user to the CourseClassView listing all the students
    • Add a button to the navigation bar using the "Add" system image that will allow the user to see a list of all available classes
    • When that button is tapped, display a modal sheet that contains the CompleteClassListView, which lets the student potentially enroll in a new class
  • CoreDataModel updates:
    • Add the StudentEntity to the xcdatamodeld file
    • Make a Student+CoreData.swift file like the other extensions from the lecture
    • Update the CourseClass+CoreData file to properly initialize the student properties in BOTH methods (in convertToManagedObject() you should add the existing students to the entity!)
    • Update the following in ClassAssignmentsCoreDataModel:
      • Add the student code to emptyDB() and loadAllDatabaseData() (including loadStudentDatasetFromJSON())
      • Add a method to get a student for a given JHED String
      • Add 2 assign methods - one to assign students to classes, and another to assign grades to students. You'll use the first method here to help enroll students in classes
  • ContentView.swift
    • Add a NSFetchRequest for the StudentEntity
    • Update the Tabview to use the EnrolledCourseClassView

Assignment Submission

  • Zip up the Xcode project folder. This should be the folder that has the .xcodeproj file inside it.
  • Rename the zip to LastName_Homework#.zip
  • Upload the zip to the Programming Assignment 3 area on Blackboard BEFORE 11:59 PM ET on the day it is due (Sunday of Week 3).
  • Be sure to test your app before submitting, so you know it works as intended!

Assignment Rubrics

In addition to the normal grading criteria stated in the syllabus, you will be graded on:

  • Proper additions to the database for the Student, including relationships
  • Proper Student extension code
  • Updates to the ClassAssignmentsCoreDataModel
  • Updates to the StudentView and the CoreDataView, including enroll in a class functionality
  • Proper save/load of values into UserDefaults
